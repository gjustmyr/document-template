# Multimedia Systems Project
**CpET 245 – Multimedia System**

## Project Overview
Students will work in groups to design and develop a website for a real client. The goal of this project is to apply multimedia principles in creating an engaging, informative, and visually appealing website.

---

## Project Value
**Total Points: 100**

---

## Group Requirements
- Each group must consist of **five (5) members**
- Collaboration and equal participation are expected from all members

## Client Requirement
- Each group must find a **real client** (individual, small business, organization, etc.)
- The chosen client must **not have an existing website**
- The group must coordinate with the client to gather necessary information and content

## Project Task
- Develop a **website** (not a system or web application)
- The website should focus on:
  - **Marketing/Promotion** of the client
  - **Informational content** (e.g., services, background, contact details)

## Project Objective
The purpose of this project is to:
- Apply multimedia concepts such as design, layout, graphics, and user experience
- Create a functional and visually appealing website
- Practice real-world client interaction and requirement gathering

## Expected Output
- A fully functional website
- Organized and visually appealing design
- Relevant and accurate content based on client information
- Complete project documentation following the provided template

---

## Website Specifications
Your website must include:
- **Multiple pages recommended** (e.g., Home, About, Services/Products, Contact) to avoid cluttered content
- **Responsive design** that works on different screen sizes
- **Multimedia elements**: Images, graphics, icons, and appropriate visual content
- **Functional navigation** between pages
- **Contact information** properly displayed
- **Clean, professional design** that reflects the client's identity

## Technology Requirements
- **HTML5** for structure and content
- **CSS3** for styling and layout
- **JavaScript** for interactivity (optional but recommended)
- All code must be original or properly attributed
- Website must be functional and tested across major browsers

## Documentation Requirements
Submit a comprehensive project documentation following the provided template that includes:
- Introduction and objectives
- Client/stakeholder information
- Requirements gathering process with evidence
- Website description and features
- Design and multimedia elements explanation
- Development tools used
- Implementation process
- Challenges and solutions
- Recommendations for future improvements
- Group member roles and contributions
- Final output (screenshots and/or live link)
- Proper references and citations
